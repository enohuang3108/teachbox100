# 重寫骰子 Lottie：六面點數正確、風格不變

## 目標
每個 dice-N 動畫落定時是一顆「真正的骰子」：頂面 = N，兩個側面 = N 的合法鄰面，
六面點數都用標準點陣、就定位、全程穩定（不再最後才換、不再 staggered pop-in）。
風格 100% 沿用現有素材（白底面、點形狀、陰影、光影）。

## 方法
1. [x] 反推三個可見面的點陣座標（從 6_5/5_2/3_5 的點中心）
   - 頂面 corners TL(-43.8,-22.9) TR(39.1,-25.5) BL(-38.7,18.5) BR(44.2,15.9)
   - 左面 corners TL(-91.3,53.3) TR(-32.6,82.6) BL(-91.3,125.1) BR(-32.6,154.4)
   - 右面 corners TL(32.6,82.6) TR(91.3,53.3) BL(32.6,154.4) BR(91.3,125.1)
2. [x] 抽出單顆點 path（v/i/o）與填色，做為可重用的點 sprite（每面各一種傾斜）
3. [x] 找出三個白底面（base-r-22/8/7）形狀「定形」的幀；決定 LAND_FRAME 並凍結成最終形
4. [x] 產生器：每個 dice-N
   - 凍結三個 hero 白底面、ip=LAND_FRAME、op=80
   - 隱藏所有烤死數字面在落定後（op<=LAND_FRAME）
   - 注入我畫的點：頂面=N、左面=A、右面=B（雙線性內插放點，沿用點形狀/色）
   - A,B = N 的合法鄰面（對面和為 7、不重複）
   - 保留 0~LAND_FRAME 的翻滾
5. [x] 逐幀渲染驗證：落定前翻滾、落定後三面正確且穩定、六個數字都對
6. [x] 接回（Dice.tsx 已用 path）遊戲（Dice.tsx 已用 path 載入），調 ROLL_MS

## 骰子鄰面（對面和為 7）
1↔6, 2↔5, 3↔4。每個 N 取兩個合法鄰面當左右側。
